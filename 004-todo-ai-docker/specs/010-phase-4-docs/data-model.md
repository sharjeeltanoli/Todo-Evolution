# Data Model for Phase IV Documentation

## Entities

### Documentation Artifact

- **Description**: Represents a single piece of documentation, which can be a Markdown file or an image.
- **Fields**:
    - `name`: The name of the file (e.g., `README.md`, `architecture.png`).
    - `format`: The format of the file (e.g., `Markdown`, `PNG`).
    - `content`: The content of the file.
- **Relationships**: None.
